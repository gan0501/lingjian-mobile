## Android版本整理逻辑
我建议采用 自下而上 的整理方式：

```
┌──────────────────────────────────────────────────
───────────┐
│                    第7层：页面/屏幕 
(Screens)                  │
├──────────────────────────────────────────────────
───────────┤
│                    第6层：业务组件 (Business 
Components)       │
├──────────────────────────────────────────────────
───────────┤
│                    第5层：基础组件 (Base 
Components)           │
├──────────────────────────────────────────────────
───────────┤
│                    第4层：Hooks & 
Utils                       │
├──────────────────────────────────────────────────
───────────┤
│                    第3层：API服务
层                            │
├──────────────────────────────────────────────────
───────────┤
│                    第2层：状态管
理                      │
├──────────────────────────────────────────────────
───────────┤
│                    第1层：设计规范/主题系统  ← 先从这
里开始      │
└──────────────────────────────────────────────────
───────────┘
```
### 整理顺序：
阶段 内容 说明 
第1阶段 设计规范/主题系统 统一颜色、字体、间距、圆角、阴影 
第2阶段 状态管理 Zustand stores 整理 
第3阶段 API服务层 接口封装、请求拦截 
第4阶段 Hooks & Utils 自定义钩子、工具函数 
第5阶段 基础组件 Button、Card、Modal、Input等 
第6阶段 业务组件 地图组件、项目卡片等 
第7阶段 页面/屏幕 各功能页面
